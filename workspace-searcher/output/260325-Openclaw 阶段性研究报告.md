# Openclaw 阶段性研究报告

## 执行摘要

OpenClaw 是2026年现象级的开源AI Agent框架，由奥地利程序员Peter Steinberger创建，前身为Clawbot（2025年11月发布）。截至2026年3月，GitHub Star数已突破**315K**，超越React和Linux成为GitHub历史上最受欢迎的开源软件项目[1]。该项目实现了AI从"能说"到"能做"的关键跨越，被誉为"AI Agent的iPhone时刻"[2]。

## Openclaw 是什么

### 核心定义
OpenClaw是一个**本地优先（Local-First）的个人AI助手框架**，它不是一个AI模型，而是一个"Agentic Harness"——将各种软件工具与用户选择的AI模型连接起来的执行框架[3]。

### 核心定位
- **本地优先**：运行在用户自己的设备上，数据完全由用户掌控
- **自主执行**：不只是回答问题，而是真正帮你完成实际工作
- **多平台集成**：通过消息平台作为主要用户界面
- **开源生态**：完全开源，拥有活跃的社区和技能生态

### 项目数据（截至2026年3月）
- **GitHub Star**: 315,000+（超越React和Linux，成为GitHub软件项目历史第一）
- **GitHub Fork**: 60,300+
- **GitHub Commits**: 19,302+
- **72小时增长**: 从9,000 Star暴涨到60,000+ Star
- **2天访客**: 200万+[4]

## Openclaw 为什么火

### 1. 开源与社区生态
OpenClaw的爆发式增长得益于其**完全开源**的特性和**活跃的社区生态**。项目在GitHub上的惊人表现：
- **60天超越React**：从第一行代码推送到GitHub，到超越Facebook用十年时间打造的React框架，仅用了不到60天[5]
- **超越Linux**：成为GitHub软件项目星标历史第一，超越了Linux过去30年的纪录[6]
- **社区贡献**：超过5,000个Pull Request，活跃的开发者社区持续贡献

### 2. 解决了AI落地的"最后一公里"
2025年是AI大模型百花齐放的一年，但大多数AI使用方式仍停留在"聊天框里打字"。OpenClaw做了一件关键的事：**给AI接上了操作系统的权限**[7]。

**从"能说"到"能做"的跨越**：
- **ChatGPT**：像一个"缸中之脑"——很聪明，但被困在浏览器里
- **OpenClaw**：给AI装上了"手"和"脚"，直接帮你把事情办了

### 3. Skills（技能插件）生态
真正让OpenClaw形成飞轮效应的，是**Skills生态**。Skills可以理解为OpenClaw的"APP Store"：
- **开发友好**：一个SKILL.md文件 + 几个Python脚本，就能创建一个完整的技能插件
- **能力市场**：技能可以被全球开发者使用，形成全新的"能力市场"
- **商业入局**：2026年2月，知名AI营销产品营赛AI宣布加入OpenClaw社区，成为首个系统性贡献Skills的商业团队[8]

### 4. 技术趋势的完美契合
- **AI Agent成熟**：2025年下半年，Agent从概念走向落地，用户心智已经准备好
- **本地化需求**：数据隐私和安全意识增强，本地部署成为刚需
- **多模型支持**：支持25+模型提供商，包括国际主流和国产模型

## Openclaw 核心架构

### 四层架构设计[9]
```
┌─────────────────────────────────────────────┐
│ Channel Layer (通道层)                      │
│ WhatsApp / Telegram / Slack / Discord / ... │
├─────────────────────────────────────────────┤
│ Gateway (网关控制面)                         │
│ WebSocket ws://127.0.0.1:18789              │
│ 状态管理 / 执行编排 / 会话路由                 │
├─────────────────────────────────────────────┤
│ LLM Brain (大模型层)                        │
│ Claude / GPT-4o / Qwen / Llama / Ollama     │
├─────────────────────────────────────────────┤
│ Skills Toolbox (技能工具箱)                  │
│ Shell / Browser / File / Canvas / Voice     │
└─────────────────────────────────────────────┘
```

### 通讯渠道（20+集成）
WhatsApp、Telegram、Discord、Slack、Signal、Teams、Matrix、IRC、LINE、Zalo、iMessage、Google Chat、Mattermost、Feishu (飞书)等20+平台。

### 模型提供商（25+支持）
- **国际主流**：Anthropic (Claude)、OpenAI (GPT-4o)、Mistral、Together
- **开源/本地**：Ollama、OpenRouter
- **云服务**：AWS Bedrock、GitHub Copilot
- **中国厂商**：GLM (智谱)、Qwen (通义千问)、Moonshot (月之暗面)、MiniMax



### Agent Loop（代理循环）
OpenClaw的核心执行流程分五个阶段：
1. **Initial Request** — 验证参数，解析会话，持久化元数据
2. **Agent Command** — 解析模型，加载技能，调用运行时
3. **Embedded Runtime** — 通过per-session和全局队列串行化执行
4. **Event Streaming** — 桥接工具事件、助手增量、生命周期事件
5. **Completion** — 监控runId，返回状态/耗时/错误信息

**关键机制**：
- **队列化执行**：按会话键串行化，防止工具/会话冲突
- **Prompt组装**：基础模板 + 技能 + 引导上下文 + 运行时覆盖
- **超时控制**：默认30秒等待，600秒运行时间上限

### 记忆系统
**双层Markdown记忆架构**：
- **日志层**：`memory/YYYY-MM-DD.md` - Append-only，每日上下文
- **长期层**：`MEMORY.md` - 持久事实/偏好/决策

**智能检索**：向量嵌入语义搜索 + BM25关键词匹配，支持时间衰减和MMR重排序。存储后端为SQLite。


## Openclaw 能做什么

### Skills（技能系统）
Skills是OpenClaw的核心扩展机制，类似于操作系统的应用程序。目前已有丰富的技能生态：

#### 核心技能类型
1. **文件操作**：读写、编辑、管理文件系统
2. **网络搜索**：Tavily、SerpAPI、微信公众号搜索等
3. **代码开发**：代码生成、调试、版本控制
4. **数据分析**：数据处理、可视化、报告生成
5. **自动化任务**：定时任务、工作流自动化
6. **多媒体处理**：图像、音频、视频处理

#### 技能开发
- **低门槛**：只需SKILL.md文件 + Python脚本
- **标准化**：统一的技能接口和文档规范
- **可分享**：通过ClawHub.com分享和发现技能

### Agents 定制
OpenClaw Agents可完全基于个人喜好定制，通过配置文件定义：

| 文件               | 说明                                 |
| ------------------ | ------------------------------------ |
| **SOUL.md**        | Agent的性格、说话风格、行为准则（界限） |
| **USER.md**        | 用户信息、习惯、偏好                   |
| **AGENTS.md**      | Agent的工作指令和任务说明              |
| **MEMORY.md**      | AI的长期记忆和学习内容                 |
| **HEARTBEAT.md**   | 定期检查和维护任务                     |
| **IDENTITY.md**    | Agent的名称、氛围、表情符号            |
| **TOOLS.md**       | 当地工具和规范的说明                   |

**记忆功能**：可以记住每一次对话，之后回溯，实现真正的个性化服务。

### 核心功能



#### 1. 定时任务（Cron Jobs）
内置完整调度系统，支持三种类型：
- **一次性任务**（at）：在指定时间执行一次
- **固定间隔**（every）：按固定时间间隔重复执行
- **Cron表达式**：使用标准的Cron表达式调度

#### 2. 多模型路由
OpenClaw支持在同一工作流中运行多个AI模型，不同的任务需要不同程度的推理和速度，系统可以智能选择最合适的模型[10]。

#### 3. 主动调度
通过Heartbeat + Cron机制，Agent可以主动执行任务，无需用户手动触发。

#### 4. 安全沙箱
提供企业级安全沙箱，确保AI操作的安全性，防止恶意行为。

### 生态系统
OpenClaw已形成完整的生态系统，包括多个衍生项目：
- **IronClaw**：企业级版本，高活跃度，24小时内处理50个PR
- **LobsterAI**：v0.2.0重大版本发布，支持Skill Store技能商店
- **Nanobot**：港大开源的超轻量级版本，4000行代码打造最轻量OpenClaw平替
- **PicoClaw**：边缘场景优化，PR吞吐量生态最高
- **ZeptoClaw**：安全冲刺版本，5/8 PR为安全修复[11]

**效率对比**：传统人工搜索需要2-3小时的任务，OpenClaw可在5-10分钟内完成，效率提升90%以上

**效率对比**：传统PPT制作需要4-6小时，OpenClaw可在30分钟内完成初稿，效率提升85%

### 4. 商业应用案例

## 挑战与风险

### 技术挑战
1. **安全风险**：2026年1月底，OpenClaw社区爆发了名为"ClawHavoc"的安全危机，AI被赋予过高系统权限可能导致安全问题[15]
2. **企业集成**：面对无API的遗留系统（Legacy System），基于代码生成的Agent可能"有力使不出"
3. **稳定性**：长流程持续高负载对系统稳定性要求高

### 市场风险
1. **监管风险**：国内多部门发布其安全应用风险提示，概念股开盘集中走弱
2. **竞争加剧**：AI Agent赛道入局者增多，行业竞争持续加剧
3. **需求不确定性**：市场对AI智能体的实际需求存在不及预期的潜在风险

### 生态风险
1. **碎片化**：多个衍生项目可能导致生态碎片化
2. **维护压力**：快速增长带来的代码质量和维护压力
3. **商业化平衡**：开源项目如何平衡商业化和社区发展
